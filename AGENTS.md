# AGENTS.md

## 项目目标

本项目是《AI客服系统 PRD》的高保真交互原型。所有代码生成、页面设计、组件命名、交互逻辑都必须优先服从 PRD，不允许自由扩展核心业务逻辑。

## 三端产品结构

本项目必须拆分为三端：

1. APP用户侧
2. 租户后台侧
3. 平台管理侧

页面右上角必须提供「端切换器」，用于切换当前访问端。

切换端后：
- 菜单必须变化；
- 页面必须跳转到该端首页；
- 数据权限必须变化；
- 不属于当前端的功能不能展示。

## 功能归属

### APP用户侧

只能包含：
- 首页
- AI客服
- 我的订单
- 售后服务
- 门店服务
- 课程/知识付费
- 大健康服务
- 我的

不得出现：
- 租户管理
- 商家管理
- 模型配置
- Prompt管理
- 知识库审核
- 质检后台
- 全平台数据看板

### 租户后台侧

只能包含本租户/本商家/本门店相关功能：
- 工作台
- 人工客服工作台
- 会话管理
- 工单中心
- 知识库管理
- AI机器人配置
- 商品/课程/门店客服资料
- 风控与大健康合规
- 质检中心
- 数据分析
- 客服团队管理
- 租户设置

不得出现：
- 全平台租户管理
- 全局模型供应商配置
- 全局Prompt管理
- 其他租户数据
- 全平台运维监控

### 平台管理侧

只能包含平台级功能：
- 平台总览
- 租户管理
- 商家管理
- 套餐与计费
- 平台知识库
- 全局Prompt管理
- 模型配置
- RAG与向量库监控
- 全局风控中心
- 平台客服工作台
- 质检与评测中心
- 运维监控
- 系统设置

不得出现：
- C端购物页
- C端个人订单页
- C端个人售后页
- 商家私有客服处理页面，除非是平台兜底视角

## 语言要求

- 所有用户可见页面文案必须使用中文。
- 所有状态名、按钮名、菜单名、表格列名、弹窗标题必须使用中文。
- Mermaid 或代码关键字可以保留英文，但页面展示内容必须中文。
- 不要出现 Created、Processing、Closed、Human_Queue 等英文状态展示。

## 业务边界

AI客服系统必须体现：
- 多租户
- 商家
- 门店
- 渠道
- 业务线
- 用户身份
- AI问答
- 人工客服
- 工单
- 知识库
- RAG
- 工具调用
- 风控
- 质检
- 数据看板
- 知识反哺

## 禁止扩展

不得实现以下能力：
- AI自动退款审批
- AI自动改价
- AI自动发券
- AI自动诊断疾病
- AI给出用药建议
- AI承诺治疗效果
- 跨租户查询数据
- 跨商家查询订单
- 无权限查看用户隐私

## 技术规范

- 使用 React + TypeScript + Tailwind CSS。
- 使用 mock 数据，不连接真实后端。
- 所有 mock 数据集中放在 `src/data/mockData.ts`。
- 页面必须组件化，避免全部写在一个文件。
- 代码必须能本地运行。
- 不允许 TypeScript 报错。
- 不允许控制台报错。

## 自检要求

每次提交前必须自检：
- 三端入口是否清晰；
- 三端菜单是否隔离；
- 页面是否完整；
- 中文文案是否完整；
- 状态名是否中文；
- 核心流程是否闭环；
- 是否和 PRD 一致；
- 是否存在自由发挥；
- 是否能运行；
- 是否没有 TypeScript 报错；
- 是否没有控制台报错。
